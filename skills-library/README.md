# Skills Library

> **Layer:** Framework Root — Shared across all profession variants  
> **Purpose:** Universal skill templates and generation prompts

---

## What Is This?

The Skills Library contains **universal skill templates** — reusable AI capability profiles that work across any profession variant. Each skill includes a `generation_prompt.md` that the user runs in their AI to produce their personalised skill file.

Skills are part of the **Extension System** (see `PROFESSION_SPEC.md §12`). They live in a `skills/` sidecar directory next to the 7-layer OS — outside the layer model, but loaded by the AI alongside the main OS bundle.

## How It Works

```
skills-library/                         ← Framework root (this directory)
└── writing-voice/
    ├── README.md                        ← What this skill enables
    └── generation_prompt.md             ← Meta-prompt to generate the skill

references/coach-os/skills/             ← Per-profession skill output
└── writing_voice.md                    ← Generated by running the prompt above
```

1. **User selects a skill** from this library
2. **User runs the `generation_prompt.md`** in their AI, providing the required inputs (e.g., 3 writing samples)
3. **AI produces** a complete, personalised skill file
4. **User saves** the output to `skills/` in their reference folder
5. **Bundle script** includes `skills/` in `skills_compiled.md`
6. **User uploads** `skills_compiled.md` alongside the main OS bundle

## Adding a New Skill to the Library

1. Create a new directory: `skills-library/{skill-name}/`
2. Add `README.md` — what the skill enables, who it's for, how to use it
3. Add `generation_prompt.md` — the meta-prompt that produces the personalised skill file
4. For each reference implementation, add a blank `{skill_output_filename}.md` file in its `skills/` folder (e.g., `references/coach-os/skills/writing_voice.md`)
5. Update this README table
